# Full Stack Deployment Guide

Deploy your AIMI application on **Vercel** (frontend) + **Render** (backend).

---

## Frontend: Vercel Deployment

### 1. Deploy to Vercel

**Option A: GitHub (Recommended)**

1. Push code to GitHub
2. Go to [vercel.com](https://vercel.com)
3. Click "New Project" → Select your repo
4. Set **Root Directory** to `frontend`
5. Click "Deploy"

**Option B: CLI**

```bash
npm install -g vercel
cd frontend && vercel
```

### 2. Set Your Backend URL

After deploying, configure the environment variable:

1. Vercel Dashboard → Your Project → **Settings**
2. Go to **Environment Variables**
3. Add:
   - **Name:** `REACT_APP_API_URL`
   - **Value:** `https://your-backend-url.render.com` (from Render)
   - **Environment:** Production
4. Click **Save**
5. Redeploy: **Deployments** → Latest → ⋮ → **Redeploy**

## Backend: Render Deployment

### 1. Push Code to GitHub

Render deploys directly from GitHub. Make sure your code is pushed.

### 2. Create Render Service

1. Go to [render.com](https://render.com) and sign up
2. Click **New +** → **Blueprint**
3. Connect your GitHub repo
4. Click **Connect**
5. Render will auto-detect `render.yaml` and deploy

### 3. Configure Environment Variables

During deployment, Render will show environment variables from `render.yaml`:

- **SECRET_KEY** — Auto-generated by Render (keep it)
- **DEBUG** — Set to `False` (keep it)
- **ALLOWED_HOSTS** — Already set to `*.render.com,localhost`
- **CORS_ALLOWED_ORIGINS** — **UPDATE THIS**:
  - Replace `https://your-vercel-frontend.vercel.app` with your actual Vercel URL
  - Your Vercel URL format: `https://your-project-name.vercel.app`

### 4. Database Setup

Render `render.yaml` includes a **free PostgreSQL** database that's automatically configured. No action needed!

### 5. Get Your Backend URL

After deployment succeeds:

1. Go to Render Dashboard
2. Click your backend service (named **aimi-backend**)
3. Copy the **URL** (e.g., `https://aimi-backend-xxxxx.render.com`)
4. Use this URL in Vercel's `REACT_APP_API_URL` setting (step 2 above)

### 6. Run Database Migrations

Render auto-runs migrations on deploy. If needed, you can manually run:

1. In Render Dashboard, click **aimi-backend**
2. Go to **Shell** tab
3. Run:

```bash
python backend/manage.py migrate
python backend/manage.py createsuperuser  # Optional: create admin
```

## Verify Everything Works

1. **Backend health check:** Visit `https://your-backend-url.render.com/api/courses/` (should return API response)
2. **Frontend loads:** Visit your Vercel URL (should load without API errors)
3. **CORS works:** Open DevTools Console and check for CORS errors (there should be none)

```bash

# Terminal 2: Backend

```

Automatically uses `localhost` via `.env.local`

## Troubleshooting

**"Connection refused" errors?**

- Wait 1-2 minutes for Render services to fully start
- Check CORS_ALLOWED_ORIGINS in Render dashboard

**"Database doesn't exist"?**

- Go to Render Dashboard → aimi-backend → Logs
- Check if migrations ran (should see "Running migrations")

**"Static files not loading"?**

- Render auto-runs `collectstatic`
- Rebuild if needed: Render Dashboard → aimi-backend → Manual Deploy

**Frontend won't connect to backend?**

- Verify `REACT_APP_API_URL` is set correctly in Vercel
- Check Render backend is running (green status in Render Dashboard)
- Check CORS_ALLOWED_ORIGINS includes your Vercel URL
